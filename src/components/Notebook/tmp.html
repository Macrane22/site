Visualizing Congressional Polarization through Network Analysis¶

From the 93rd through the 114th Congress¶

Network analysis is an ideal lens through which to understand patterns of congressional behavior. It's simple to conceive of a network in which the nodes are lawmakers and the edges represent some kind of interaction between them. Once this structure is in place, it's possible to employ measures of centrality, density, and community detection to obtain a holistic view of partisanship and polarization, among other things.

In this notebook, I use a dataset of bill sponsorships to build a series of networks over time, where each network represents one Congress. The data contains all sponsors and cosponsors of bills between the 93rd and 114th Congresses, which span the years between 1973 and 2016. I connect two congresspeople if they cosponsored a bill, and I weight that connection by the number of bills they appear on together. By looking at patterns of cosponsorship in the House, we can get a sense of how congressional behavior and polarization have shifted over time.

In this notebook, I employ community detection (using the Louvain algorithm to split the congressional network into communities.

I use networkx to build and analyze the network, community to run the Louvain algorithm, and matplotlib for visualization.

About the data¶

This data is downloaded from Prof. James Fowler's personal webpage at http://jhfowler.ucsd.edu/cosponsorship.htm. It originates from the US government's Thomas database, via a scrape by Govtrack. Both the Thomas database and the Govtrack scraping service are now deprecated, so this data extends only through the 114th Congress, which adjourned in 2016. I reshaped the data into a list of lawmaker nodes and lawmaker-lawmaker weighted edges, and created a networkx graph out of them. I'll make that code available on Github.

In addition, because the Fowler dataset lacks the lawmakers' parties and states, I extracted congressional metadata from Govtrack's US Congress Legislators dataset.

The original data contains both House and Senate bills; I filter out the latter and include only the House in this analysis.

def assemble_network(nodes_path, edges_path):
    network = nx.Graph()

    node_df = pd.read_csv(nodes_path)
    nodes = node_df.to_dict('records')
    node_attrs = nodes[0].keys()
    for node in nodes:
        node_id = node['id']
        network.add_node(node_id)
        for attr in node_attrs:
            network.nodes[node_id][attr] = node.get(attr)
    
    edge_df = pd.read_csv(edges_path)
    edges = edge_df.to_dict('records')
    for edge in edges:
        network.add_edge(edge['source'], edge['target'], weight=edge.get('weight', 1))

    main_network = network.subgraph(max(nx.connected_components(network), key=len))
    main_network = convert_node_labels_to_integers(main_network)
    return node_df, edge_df, main_network


def louvain_partition(network, resolution=1.):
    """Runs the Louvain modularity algorithm on a network and returns the algorithm's best partition."""
    dendrogram = generate_dendrogram(network, random_state=42, resolution=resolution)
    # Get a list of partition levels
    partitions = []
    for level in range(len(dendrogram)):
        partitions.append(partition_at_level(dendrogram, level))
    # For the moment, just take the "best" Louvain partition
    best_partition = partitions[-1]
    return best_partition

def homogeneity(network, partition, col='party'):
    enc = LabelEncoder()
    partition_sorted = [i[1] for i in sorted(partition.items())]
    data_sorted = [i[1][col] for i in sorted(network.nodes(data=True))]
    data_enc = enc.fit_transform(data_sorted)
    homog = homogeneity_score(partition_sorted, data_enc)
    return homog


def mean_conductance(network, partition):
    conds = []
    for comm_id in set(partition.values()):
        comm_nodes = [node for node, comm in partition.items() if comm == comm_id]
        node_n = len(comm_nodes)
        sg = network.subgraph(comm_nodes)
        # Calculating conductance on a single-node community will result in a zero-division error
        if node_n < 2:
            continue
        else:
            cond = np.round(nx.conductance(network, sg, weight='weight'), 3)
            conds.append(cond)
    mean_cond = np.mean(conds)
    return mean_cond if not math.isnan(mean_cond) else 0.

III. Analysis¶

1. First look: Weighted edge degree¶

First we should understand the distribution of edges across the network: How typical is it for one lawmaker to sign a bill with another lawmaker, ignoring for now their party? We already know that the networks are very dense; thus, the distribution of node degrees will be skewed heavily towards 100%: Most lawmakers have cosponsored at least one bill with virtually every one of their peers. The weighted edge degree over time, however, will give a more nuanced picture of how "social" lawmakers tend to be with one another.

2. Intra-Community Homogeneity¶

We'd like to understand the extent to which these communities consist of representatives from a single party. In the highly polarized political environment in the U.S., we'd expect the communities to map fairly well to the Democratic Republican parties. Moreover, it would be interesting to see the extent to which this polarization has shifted over time.

We can visualize the composition of the communities by plotting a grid of squares to represent each community in a given Congress, where one square represents one lawmaker (blue for Democrats, red for Republicans, and yellow for everyone else).

def community_party_summary(partition, network):
    comm_ids = [i for i in range(len(set(partition.values())))]
    parties_by_comm = []
    for comm_id in comm_ids:
        comm_nodes = [node for node, comm in partition.items() if comm == comm_id]
        sg = network.subgraph(comm_nodes)
        _parties = [data['party'] if data['party'] in named_parties else 'other' for _, data in sg.nodes(data=True)]
        _parties = [party_mapping[p] for p in _parties]
        parties_by_comm.append(_parties)
        
    return parties_by_comm
        

def plot_comm_grids(congress, parties_by_comm):
    ncols = 15
    ncomm = len(parties_by_comm)
    fig, axes = plt.subplots(1, ncomm, figsize=(ncomm, 1))
    for ax, comm in zip(axes.flat, parties_by_comm):    
        n = len(comm)
        nrows = int(np.ceil(n / ncols))
        comm = np.asarray(comm)
        grid_n = nrows * ncols 
        blanks = grid_n - n
        for i in range(blanks):
            comm = np.append(comm, 3)
        comm = np.reshape(comm, (nrows, ncols))
    
        ax.get_xaxis().set_visible(False)
        ax.get_yaxis().set_visible(False)
        ax.spines['right'].set_visible(False)
        ax.spines['top'].set_visible(False)
        ax.spines['bottom'].set_visible(False)
        ax.spines['left'].set_visible(False)
        ax.set_anchor('N')
        ax.imshow(comm, cmap=cmap)
        ax.set_aspect(1)

    title = fig.suptitle('Congress: {}'.format(congress), fontsize=7);
    title.set_y(1.05)#(1.02)
    title.set_ha('right')
    return axes

3. Polarization: Mean Conductance¶

We can see that intra-community homogeneity has increased. But what of the connections between communities? It seems likely that lawmakers will share fewer bill cosponsorships as their clusters become more homogeneous.

Conductance is the ratio of the edge weight linking two sets of nodes with the total edge weight of nodes in the smaller of the two sets. Conductance near 0 indicates that the given community is highly separate from the rest of the network. In the typical case, a community's conductance measures its connection to the rest of the network. We can estimate conductance for the network as a whole by taking the mean conductance across all communities.

3. Bridge lawmakers: Betweenness centrality¶

Given the increase in polarization between congressional communities, it would be interesting to know whether there are lawmakers who act as bridges between the sides. Betweenness centrality is an ideal metric for such a use case: It measures the extent to which a node lies on the shortest path between all pairs of other nodes. Nodes with high betweenness centrality tend to be important in their network, because information flowing from one part of the network to another is likely to pass through them.

For our use case, though, we need to make one tweak: We care only about edges that run between communities. Intracommunity edges may be interesting in their own right, but they won't tell us much about lawmakers who bridge America's partisan divide. Luckily it's not difficult to alter networkx's implementation of betweenness for our purposes.

Betweenness centrality is a computationally intensive algorithm, because it requires the calculation of all shortest paths between each pair of nodes (although networkx allows you to esimate it from a sample of nodes). Fortunately, these networks are small enough, at < 500 nodes each, for it to be feasible here.

In [207]:
from networkx.algorithms.centrality.betweenness import _accumulate_basic
from networkx.algorithms.centrality.betweenness import _rescale
from networkx.algorithms.centrality.betweenness import _single_source_dijkstra_path_basic


def intercommunity_betweenness_centrality(nw, part):
    all_nodes = nw.nodes
    betweenness = dict.fromkeys(all_nodes, 0.0)
    for node in all_nodes:
        comm = part[node]
        nodes_diff = [n for n in all_nodes if part[n] != comm]
        endpoints, paths, sigma = _single_source_dijkstra_path_basic(nw, node, weight='weight')        
        # Filter to only paths where the nodes are not in the same community:
        paths = {k: (v if k in nodes_diff else []) for k, v in paths.items()}
        # Add 1.0 to all sigmas to avoid zero division error:
        sigma = {k: (v + 1.0 if k in nodes_diff else 1.0) for k, v in sigma.items()}
        betweenness = _accumulate_basic(betweenness, endpoints, paths, sigma, node)
    betweenness = _rescale(betweenness, len(nw), normalized=True, directed=nw.is_directed())
    return betweenness
